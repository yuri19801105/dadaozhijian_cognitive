# ADR-0004: 六合 → 七星 的"态势供给规划"关系

## 状态

已接受

## 背景

六合(6,空间感知与上下文建模)与七星(7,动态规划与任务调度)在原始蓝图中被并列列出,但二者如何协同未明。若七星仅依子目标内在属性排序,会脱离真实环境约束,沦为"纸上谈兵"。

## 决策

确立 **六合 → 七星 为"地图 → 规划"的供给关系**:七星在规划时**必须消费**六合产出的 `context_vector[6]` 与 `situation_map`,并据此执行以下四项机制:

1. **可行性剪枝** — 排序前读 `context_vector[6]`(东/南/西/北/上/下 = 可达性/约束/层级),剔除上下文不可达的排序。
2. **动态优先级** — 以六合空间相关性/邻近度给子目标打分;随六合态势更新而重排(对应斗柄随季转向的时序演化)。
3. **上下游锚定** — 六合"上/下"维度(层级)指示目标的上游依赖/下游产物,据此定先后,而非平铺。
4. **接地(grounding)** — 七星产出的每个子目标携带六合坐标系锚点,使计划可被十方执行层直接消费。

## 理由

- 六合补全"空间感/情境感",使认知从内在循环扩展到知行合一。
- 七星据此获得"在真实约束下动态规划与预见"的能力,而非任意排序。
- 将二者关系显式化为供给契约,避免实现时各自定义接口导致耦合错乱。

## 影响

- `qixing.plan(subgoals, context_vector)` 的契约中,`context_vector` 为必填参数(见 `docs/architecture.md` 第 2、3 节)。
- 六合模块须保证 `context_vector` 覆盖全部已知约束,否则七星剪枝可能误删可行路径。
- 后续若引入真实环境观测,仅扩展六合观测端,七星消费契约不变。
