# ADR-0009: 十方执行器算子映射决策

## 状态

已采纳

## 背景

十方(Executor)作为认知模型第 10 层的多模态 I/O 总接口,需要将七星规划的有序算子链转化为具体的文本输出动作,完成"认知→行动"闭环。

## 决策

### 1. 算子 → 动作映射

| 算子 | 动作语义 | 输出模板 |
|------|---------|---------|
| 乾(创造) | 生成新内容 | `[N] 创造: {raw_input}` |
| 坤(承载) | 存储/记录九宫状态 | `[N] 承载: 已占用 {occupied} 格` |
| 震(启动) | 触发信号 | `[N] 启动: 处理 {raw_input}` |
| 巽(渗透) | 扩散/整合 | `[N] 渗透: 整合输入到九宫` |
| 坎(冒险) | 风险提示/假设测试 | `[N] 冒险: 尝试假设——长度 {N} 字节` |
| 离(明辨) | 判断结果 | `[N] 明辨: 输入判定为 奇数/偶数` |
| 艮(停止) | 终止分支 | `[N] 停止: 结束分支` |
| 兑(交流) | 输出表达 | `[N] 交流: {raw_input}` |
| 未知 | 回退 | `[N] 未知算子` |

### 2. 输出格式

- 多行文本,每行对应一个算子
- 行首 `[N]` 是执行序号
- 最后一行无尾随换行

### 3. 输出类型

`String` — 可直接通过十方多模态接口发送

### 4. 反馈机制

执行器不直接修改九宫(因 Mojo 1.0b2 参数传递限制),但返回的输出字符串可由调用者通过 `ws.hold()` 持久化到工作记忆中。

## 影响

- 完成"十方"的完整闭环:输入(M1/M2) → 认知(M3–M7) → 输出(M10)
- 全认知周期延迟 ≈ 732ns(调度100ns + 态势12ns + 规划270ns + 执行350ns)  *(勘误 2026-07-10: 原记 786ns/规划280ns/执行366ns, 已随各模块实测校准: 规划 280→270[M7], 执行 366→350[M10]; 调度取 ~100ns[M5 Wood])*
- 下游可扩展多模态输出:当前为文本,可扩展为图像/音频/命令

## 性能基准

| 场景 | 1M 次耗时 | 单次 |
|------|----------|------|
| execute(3 算子链) | 350ms | **350ns** |
