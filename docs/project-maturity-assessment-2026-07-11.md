# 项目成熟度评估（MVP vs 生产模型）— 2026-07-11

> 评估基于 `src/` 源码与磁盘文件实际核查，非文档声称。
> 结论：**当前为 MVP / 概念验证（框架可行性验证），不是可训练、可上线的完整模型。**

## 一、框架完成度（核心模块是否全部实现）

- ✅ 脚手架层面 100%：M1 分词 / M2 正则 / M3 九宫 / M4 八卦 / M5 五行 / M6 六合 / M7 七星 / M8 可视化 / M9 太极 / M10 执行器 均有源码+测试，端到端跑通（`CognitiveCycle.run` → recall → schedule → plan → execute → feedback），116 测试全绿。
- ⚠️ "实现"= 手写确定性符号规则，非可学习组件：
  - `executor.mojo` 8 算子全是字符串模板（贴标签式输出），无真正理解/生成能力。
  - `wu_xing.mojo` "权重"为硬编码常量（依卦象→五行查表赋值），非学习参数。
  - `workspace.mojo` "注意力 Tensor[9]" 实为 `List[Int]` 类比，无真实注意力机制（Mojo 无 Tensor 类型）。
  - `qixing.mojo` 为确定性 DP 排序，非学习排序。
- 结论：框架骨架齐备自洽 ✅；认知智能本身为占位级 ⚠️。

## 二、训练流程是否产出可用模型权重

- ❌ 完全没有模型训练流程，无任何模型权重。
- `src/` 全扫描：无神经网络/反向传播/梯度/张量/嵌入/可学习参数。
- 仅有的 `train()`：`bpe.mojo`/`regex.mojo` 为**分词器词汇归纳**（字节对合并/GPT-4 切分），产出内存 vocab/merges，且**不落盘**。
- 磁盘扫描 `*.bin/*.pt/*.safetensors/*.ckpt/*.onnx/*.model/*weights*/*checkpoint*/*.npz/*.npy`：**零文件**。
- 无 `http/api/llm/openai/endpoint` 真实外部调用。
- 结论：分词词汇可跑，但**不产出、不保存、不加载任何"模型权重"**。

## 三、与生产上线要求差距（数量级，非修补级）

| 维度 | 当前状态 | 上线要求 | 差距 |
|---|---|---|---|
| 能力本质 | 确定性规则模板 | 真正理解/生成 | 根本性缺失 |
| 外部模型对接 | M10 不调 LLM/API | 需对接基座/自有模型服务 | 完全缺失 |
| 模型权重 | 无 | 训练+版本化+加载 | 完全缺失 |
| 性能 | 纳秒/微秒（仅查表/排序） | 真实推理吞吐/延迟 | "快"为假象 |
| 稳定性 | 116 单测，无容错/边界/监控 | 容错+边界+监控+日志 | 巨大 |
| 可扩展性 | 单进程内存态，TaijiState 不落盘，无并发/分布式 | 服务化+持久化+水平扩展 | 巨大 |
| 数据/评测 | 无数据管线、无评测集、无指标 | 数据闭环+量化评测 | 巨大 |
| 工程化 | 无 CI/容器/部署/模型版本管理 | 完整 MLOps/DevOps | 巨大 |
| 安全合规 | 无 | 隐私/安全/合规 | 未涉及 |

## 四、判断与建议

- 目标=验证框架可行性：✅ 已达标（可论证"东方认知范式→可计算流水线"工程自洽）。
- 目标=可上线模型：❌ 本质缺"模型层"。需二选一补齐：
  1. 对接外部 LLM/API（M10 占位算子→真实调用+提示词编排），先让架构"能说话"成 demo；
  2. 在 M4–M7 上训练可学习参数（引入 Mojo ML 能力或桥接 Python 训练栈并落盘权重）。
- 当前不属于"训练即上线"，也不属于"完整模型"，是**概念验证型 MVP**。
